# QR+ бот — структура контента и соответствие скрипту (RUS)

Документ связывает **скрипт бота (клиент, RUS)** с файлами и ключами в коде. По нему можно заполнять тесты и проверять контент.

---

## 1. Общая схема работы с текстами

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         ИСТОЧНИКИ ТЕКСТОВ                                │
├─────────────────────────────────────────────────────────────────────────┤
│  • authorization.py   — первое знакомство, выбор языка (без i18n)        │
│  • auth.py            — регистрация (ФИО, дата, пол, email, политика)   │
│  • keyboards/auth.py  — кнопки: Қазақ тілі, Русский язык, Принять       │
│  • main.py            — приветствие зарегистрированного, меню           │
│  • data/faq_new.py    — тексты и кнопки главного меню + lvl4, lvl5, 7, 8│
│  • data/faq_qna.py    — раздел «Вопросы и ответы» (lvl6, доставка,      │
│                         обмен/возврат, адреса, Kaspi, наличие, гифткарта)│
│  • data/faq_translations.py — казахские переводы (rus/kaz) для FAQ      │
│  • data/faq_locales.py      — локализованные тексты через _()           │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  ХЕНДЛЕРЫ: auth → main → faq. По callback (lvl4, lvl5, lvl6_*) берут    │
│  текст из faq_texts_update / qna_texts / faq_translations и показывают  │
│  кнопки из faq_lvls / qna_lvls.                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. Замечания по скрипту и тестам

- **Язык:** В скрипте «Русский язык» — в коде кнопка так и подписана (`keyboards/auth.py`). Без флажков по скрипту — в боте используются кнопки (Қазақ тілі / Русский язык).
- **Орфография:** Везде в контенте приведено к «кэшбэк» (не «кэшбек»).
- **Маркеры по ТЗ:** В блоке «Как работает кэшбэк» (lvl5) используются точки «•», не тире.
- **Кнопка «Возврат онлайн»:** В скрипте текст «Возврат онлайн (возврат денежных средств офлайн)» — в коде оставлено как в скрипте для совпадения с тестами; по смыслу там должно быть «...денежных средств **онлайн**».
- **История бонусов:** В меню есть пункт «История бонусов» (между «Мой баланс» и «Мой QR-код»); в скрипте нумерованный список 1–8 его не перечисляет — при тестах учесть отдельно.
- **Политика конфиденциальности:** URL в коде: `https://qazaqrepublic.com/ru/privacy` — как в скрипте.

## 4. Где править тексты под скрипт

| Что править | Файл |
|-------------|------|
| Первое сообщение, выбор языка | `tgbot/handlers/authorization.py` |
| Регистрация (вопросы, политика, ФИО, дата, пол, email) | `tgbot/handlers/client/auth.py` |
| Приветствие зарегистрированного и после регистрации | `tgbot/handlers/client/main.py` |
| Главное меню, баланс (lvl1), QR+ (lvl4), кэшбэк (lvl5), сотрудничество (lvl7), поддержка (lvl8) | `tgbot/data/faq_new.py` |
| Вопросы и ответы (доставка, самовывоз, Kaspi, обмен/возврат, адреса, наличие, гифткарта) | `tgbot/data/faq_qna.py` |
| Казахские переводы блоков FAQ | `tgbot/data/faq_translations.py` |
| Локализованные строки через _() | `tgbot/data/faq_locales.py` (если используются) |

По этой структуре можно пройтись по скрипту и заполнить тесты по контенту бота.
